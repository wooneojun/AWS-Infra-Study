# AWS Systems Manager (SSM)

## 1. 개요 및 통합 관리 능력

AWS Systems Manager는 AWS 리소스뿐만 아니라 온프레미스 데이터 센터의 서버까지 하나의 콘솔에서 제어할 수 있는 관리 서비스입니다.

* **통합 관리:** EC2 인스턴스와 온프레미스 서버 모두에 **SSM Agent**를 설치하여 물리적 위치에 상관없이 동일한 정책을 적용합니다.
* **단일 보고서:** **Patch Manager** 및 **Compliance** 기능을 통해 전체 인프라의 보안 패치 상태를 추적하고 통합 규정 준수 보고서를 생성합니다.

---

## 2. 관리 대상 범위 (Coverage)

SSM은 하이브리드 클라우드 환경의 모든 컴퓨팅 리소스를 관리형 노드(Managed Nodes)로 등록하여 관리합니다.

### 2.1 AWS 내부 리소스

* **Amazon EC2:** 기본 관리 대상이며, 최신 AMI에는 SSM Agent가 사전 설치되어 있습니다.
* **AWS IoT Greengrass:** 엣지 디바이스의 원격 구성 및 관리를 지원합니다.
* **Amazon ECS/EKS:** 컨테이너의 메타데이터 수집 및 내부 명령 실행에 활용됩니다.

### 2.2 하이브리드 및 멀티 클라우드

* **온프레미스 서버:** 물리 서버 및 가상 머신(VM)을 관리형 노드로 등록 가능합니다.
* **타 클라우드 VM:** Azure, GCP 등의 가상 머신도 SSM Agent 설치를 통해 AWS 콘솔에서 통합 관리할 수 있습니다.

## 3. 주요 관리 기능 범주

SSM의 기능은 크게 네 가지 영역으로 나뉘어 인프라 수명 주기를 관리합니다.

### 3.1 노드 관리 (Node Management)

실제 서버(노드)를 직접 제어하고 관리하는 핵심 기능입니다.

* **Session Manager:** SSH 키 관리나 포트 개방 없이 브라우저에서 보안 셸(Shell) 접속을 제공합니다. (보안성 극대화)
* **Run Command:** 수천 대의 인스턴스에 명령어를 동시 실행하고 결과를 수집합니다.
* **Patch Manager:** OS 및 소프트웨어 패치 작업을 자동화합니다.
* **Inventory:** 인스턴스에 설치된 소프트웨어 목록과 네트워크 정보를 수집합니다.

### 3.2 애플리케이션 관리 (Application Management)

앱 설정과 보안 데이터를 안전하게 관리합니다.

* **Parameter Store:** 암호, API 키, 라이선스 등을 안전하게 저장하고 앱에 제공합니다.
* **AppConfig:** 앱 중단 없이 동적으로 설정을 배포하고 오류 시 자동 롤백을 수행합니다.

### 3.3 운영 관리 (Operations Management)

운영 효율성을 높이고 이슈를 시각화합니다.

* **OpsCenter:** 인프라의 각종 운영 문제(OpsItems)를 한곳에서 집계하고 해결을 돕습니다.
* **Explorer:** 전체 리소스의 상태와 성능 지표를 대시보드 형태로 제공합니다.

### 3.4 변경 관리 (Change Management)

안전한 인프라 변경 프로세스를 지원합니다.

* **Automation:** 반복적인 운영 업무를 워크플로 형태로 자동화합니다.
* **Change Manager:** 인프라 변경 사항에 대한 승인 절차를 체계화하여 관리합니다.

## 4. 핵심 요약 및 시사점

| 특징 | 상세 내용 |
| --- | --- |
| **운영 효율성** | SSH 접속 없이 다수의 서버를 동시에 제어하여 관리 리소스 절감 |
| **보안 및 규정 준수** | 모든 명령 실행 내역 기록(Audit) 및 중앙 집중식 패치 관리 |
| **유연성** | 클라우드와 온프레미스를 가리지 않는 하이브리드 관리 환경 제공 |
