# Amazon EBS(Elastic Block Store)

## 1. Amazon EBS란?

클라우드에서 사용하는 가상 하드디스크(Block Storage)입니다. EC2 인스턴스가 연산(CPU/Memory)을 담당한다면, EBS는 데이터를 저장하는 SSD/HDD 역할을 수행합니다.

* **영구 스토리지:** 인스턴스의 생명주기와 별개로 데이터를 영구적으로 보존할 수 있습니다.
* **유연한 조절:** 사용량과 성능을 몇 분 내에 조절할 수 있으며, 프로비저닝(할당)한 자원에 대해서만 비용을 지불합니다.

---

## 2. EBS-EC2 연결 및 운용 특징

네트워크를 통해 연결되는 독립적인 서비스이기 때문에 물리 서버의 디스크보다 유연한 운용이 가능합니다.

* **독립적 유지:** EC2 인스턴스를 종료해도 EBS 볼륨은 삭제되지 않고 유지될 수 있습니다. (인스턴스 정지 시 CPU 비용 없이 저장 비용만 발생)
* **볼륨 이동:** 부착된 EBS를 분리하여 성능이 더 좋은 다른 EC2 인스턴스에 즉시 연결할 수 있습니다.
* **다중 부착:** * 하나의 인스턴스에 여러 EBS 볼륨을 부착할 수 있습니다.
* **Multi-Attach:** 하나의 EBS 볼륨을 여러 인스턴스에 동시에 부착하는 것도 가능합니다. (특정 볼륨 타입 및 클러스터링 환경에서 활용)


* **가용 영역(AZ) 제한:** EBS는 **EC2와 동일한 가용 영역(AZ) 내에 존재**해야 합니다. 데이터 전송 속도와 지연 시간을 최소화하기 위함이며, 다른 AZ의 인스턴스에는 직접 부착할 수 없습니다.

---

## 3. EBS 볼륨 타입 및 특징

사용 목적과 요구 성능에 따라 5가지 주요 타입을 제공합니다. 현재는 가성비가 뛰어난 **gp3** 사용이 가장 권장됩니다.

### 3.1 SSD 기반 볼륨 (트랜잭션 워크로드용)

| 타입 | 이름 | 주요 특징 |
| --- | --- | --- |
| **gp3** | **범용 SSD (최신)** | 가성비가 가장 우수함. 용량과 무관하게 IOPS와 처리량을 독립적으로 조절 가능. |
| **gp2** | **범용 SSD (이전)** | 볼륨 크기에 비례해 성능이 결정됨. 작은 용량에서는 버스트 크레딧 방식을 사용함. |
| **io2 / io1** | **프로비저닝 된 IOPS** | 극도의 성능과 낮은 지연 시간이 필요한 대규모 DB 워크로드에 사용. |

> **버스트 크레딧 방식(gp2):** 평상시 성능을 비축했다가 트래픽 급증 시 크레딧을 소모하여 일시적으로 성능을 높이는 메커니즘입니다.

### 3.2 HDD 기반 볼륨 (처리량 위주 워크로드용)

| 타입 | 이름 | 주요 특징 |
| --- | --- | --- |
| **st1** | **처리량 최적화 HDD** | 로그 처리, 빅데이터 분석, 데이터 웨어하우스 등 빈번한 액세스에 적합. |
| **sc1** | **콜드 HDD** | 접근 빈도가 낮은 데이터를 가장 저렴하게 저장할 때 사용. |
| **Standard** | **마그네틱** | 이전 세대 스토리지로, 현재는 거의 사용되지 않음. |
