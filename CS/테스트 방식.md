# 소프트웨어 테스트

- 품질 보증을 위해 개발 단계별로 단위, 통합, 시스템, 인수 테스트로 나뉨

## 테스트 레벨 (개발 단계에 따른 분류)

- 단위 테스트(Unit Test) : 함수, 메소드 등 가장 작은 모듈 단위 독립적으로 테스트
- 통합 테스트(Integration Test): 단위 기능들을 결합해 모듈 간 인터페이스와 데이터 흐름이 올바른지 확인
- 시스템 테스트 (System Test) : 통합된 시스템 전체가 기능 및 비기능 요구사항을 만족하는지 실제 환경과 유사한 조건에서 테스트
- 인수 테스트 (Acceptance Test) : 사용자가 최종 요구사항을 만족하는지 확인해 제품 출시 여부 결정 (A/B 테스트 포함)

## 테스트 기법(접근 방식에 따른 분류)

- 블랙박스 테스트 : 내부 구조를 모르는 상태에서 입력값에 따른 출력 결과만 테스트
- 화이트박스 테스트 : 코드의 내부 로직, 구조를 보며 논리 경로를 테스트

## 테스트 목적 및 성격에 따른 분류

- 기능 테스트 : 기능 요구사항 준수 여부
- 비기능 테스트 : 성능, 사용성, 신뢰성, 보안성 테스트
- 회귀 테스트 : 코드 변경 후 기존 기능에 오류가 없는지 재테스트

## 실행 여부에 따른 분류

- 정적 테스트 : 코드를 실행하지 않고 문서나 코드를 리뷰하여 오류 검출
- 동적 테스트 : 프로그램을 실제 실행해 결과 확인
