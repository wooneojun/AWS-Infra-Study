[Networking] AWS Gateway Load Balancer(GWLB) 핵심 요약
1. Gateway Load Balancer(GWLB)란?
기존 클라우드 환경에서 방화벽이나 침입 탐지 시스템(IDS/IPS)과 같은 서드 파티 보안 가상 어플라이언스를 운영하려면 복잡한 네트워크 설정이 필요했습니다. GWLB는 이러한 장비들을 쉽게 배포, 확장 및 관리할 수 있게 해주는 전용 로드 밸런서입니다.

핵심 프로토콜: **GENEVE (RFC 8926)**를 사용합니다. 원본 패킷의 소스/목적지 정보를 유지한 채 보안 장비로 트래픽을 캡슐화하여 전달하기 위함입니다.

투명성: 트래픽이 보안 장비를 거치지만, 실제 송수신자는 네트워크 경로상의 보안 장비 존재를 인식하지 못하는 투명한 레이어 역할을 수행합니다.

2. 주요 설정 및 구성 요소
네트워크 매핑: GWLB가 위치할 VPC와 서브넷을 지정하며, 이는 보안 장비(대상 그룹)들이 위치한 곳과 일치해야 합니다.

IP 리스너: 특정 포트가 아닌 모든 포트와 프로토콜의 IP 패킷 전체를 수신하여 대상 그룹으로 전달합니다.

GWLB 엔드포인트(GWLBE): 트래픽이 드나드는 실제 '인터페이스' 역할을 합니다. VPC 라우팅 테이블에서 이 엔드포인트를 타겟으로 설정하여 트래픽을 GWLB로 유도합니다.

3. 주요 이점 (Benefits)
고가용성 (High Availability): 보안 장비들을 대상 그룹으로 묶어 관리하며, 특정 장비 장애 시 정상적인 장비로 트래픽을 자동 라우팅합니다.

유연한 확장성 (Scaling): 트래픽 부하에 따라 보안 어플라이언스(EC2)의 개수를 Auto Scaling을 통해 자동으로 조절할 수 있습니다.

중앙 집중식 보안: 여러 VPC의 트래픽을 하나의 보안 전용 VPC로 모아서 검사하는 '인스펙션 VPC(Inspection VPC)' 아키텍처 구현이 용이합니다.

4. 주요 사용 사례 (Use Cases)
인바운드/아웃바운드 보안: 인터넷 유입 트래픽의 악성 코드 검사 및 내부 데이터의 외부 유출 방지.

VPC 간 트래픽 검사 (East-West): 동일 리전 내 서로 다른 VPC 간 통신 시 보안 정책 준수 여부 실시간 모니터링.

전문 보안 솔루션 활용: AWS Native 보안 기능 외에 Palo Alto, Fortinet 등 시중의 전문 보안 장비를 클라우드에서 동일하게 활용하고자 할 때.
