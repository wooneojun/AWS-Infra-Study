# [Case Study] redBus: Amazon QuickSight를 활용한 데이터 플랫폼 현대화

## 1. 도입 배경 및 도전 과제

redBus는 매일 3억 2천만 건 이상의 이벤트를 처리하며 데이터 규모가 페타바이트(PB) 단위로 급증함에 따라, 기존 Apache Superset 기반 환경에서 다음과 같은 한계에 직면했습니다.

* **높은 운영 부담 (High TCO):** EC2, MySQL, Redis, RabbitMQ 등 다수의 컴포넌트를 직접 관리, 백업, 확장해야 하는 인프라 오버헤드 발생.
* **기능적 제약:** 이기종 데이터 소스 간의 결합 시각화 및 대시보드 전역 필터 기능의 부재.
* **낮은 사용성:** 대시보드 커스터마이징을 위해 JSON 수정 등 개발자 수준의 지식이 요구되어 비즈니스 사용자의 접근성이 저하됨.

---

## 2. 현대화된 아키텍처: Data Highway 2.0

redBus는 QuickSight를 핵심 시각화 도구로 도입하며 데이터 플랫폼을 서버리스 기반으로 진화시켰습니다.

### 2.1 데이터 수집 및 카탈로깅

* **다양한 소스:** 모바일 앱, 마이크로서비스, RDS(OLTP), Cassandra 등에서 실시간/배치 데이터를 수집합니다.
* **Apache NiFi:** 자체 관리형 NiFi 클러스터를 통해 복잡한 데이터를 정제, 분석 및 카탈로깅합니다.

### 2.2 저장 및 분석 레이어

* **Amazon S3:** 무제한 확장성을 가진 데이터 레이크의 핵심 스토리지로 활용합니다.
* **Apache Druid:** 실시간 처리를 위한 OLAP 스타일 데이터 스토어입니다.
* **Amazon Redshift:** S3의 정제된 데이터와 Druid의 집계 데이터를 저장하는 통합 데이터 웨어하우스(DW)입니다.

### 2.3 시각화 레이어 (QuickSight 도입)

* **통합 BI:** S3 및 Redshift와 네이티브하게 연결된 **Amazon QuickSight**를 통해 전사적 비즈니스 인사이트를 제공합니다.
* **병행 운영:** 실시간 APM(Application Performance Monitoring)을 위해 Druid와 연결된 기존 Superset을 특수 목적으로 일부 병행 사용합니다.

---

## 3. Amazon QuickSight 도입의 주요 이점

* **서버리스 아키텍처:** 인프라 관리 없이 자동 확장되어 운영 노력을 획기적으로 절감합니다.
* **SPICE 엔진:** 강력한 인메모리 엔진(SPICE)을 통해 수천 명의 사용자가 동시에 대규모 데이터를 조회해도 빠른 응답 속도를 보장합니다.
* **셀프 서비스 BI:** 드래그 앤 드롭 방식으로 코딩 없이 비즈니스 분석가가 직접 대시보드를 생성할 수 있습니다.
* **고급 분석 기능:** 행 수준 보안(RLS)을 통한 국가별 데이터 접근 제어 및 ML 기반 이상 탐지(Anomaly Detection)를 추가 인프라 없이 활용합니다.

---

## 4. 핵심 활용 사례: 매출 이상 탐지 대시보드

redBus는 데이터 플랫폼을 통해 비즈니스 리스크를 실시간으로 관리하고 있습니다.

1. **데이터 결합:** Redshift의 과거 이력 데이터와 Druid의 실시간 트래픽 데이터를 결합합니다.
2. **ML 모델 분석:** 데이터를 redBus 고유의 예측 모델에 입력하여 예상 매출과 실제 매출을 비교합니다.
3. **즉각적 시각화:** 분석 결과를 QuickSight의 꺾은선 그래프로 시각화하여, 매출 지표가 예상 범위를 벗어날 경우 비즈니스 이상 징후로 즉각 감지하고 대응합니다.

## 5. 최종 요약

| 항목 | 기존 환경 (Superset) | 현대화 환경 (QuickSight) |
| --- | --- | --- |
| **인프라 관리** | 수동 (EC2, DB 등 직접 관리) | **서버리스 (Managed)** |
| **확장성** | 구성 요소별 수동 확장 | **자동 확장 (Auto-scaling)** |
| **접근성** | 개발자 중심 (JSON 수정 등) | **현업 중심 (Self-service)** |
| **데이터 결합** | 데이터 소스 간 결합 어려움 | **네이티브 커넥터를 통한 손쉬운 통합** |
