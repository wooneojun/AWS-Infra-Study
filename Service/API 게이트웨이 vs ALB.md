## API 게이트웨이 : 보안 가드

- 인증 토큰 확인, 활동 기록, 불필요한 정보 걸러내고 적절한 마이크로 서비스로 이동

### 특징

1. **중앙집중식 라우팅** : /webhooks/order 는 하나의 서비스로 /auth/verify는 다른 서비스로 라우팅. 이를 통해 버전 관리 및 트래픽 제어 용이
2. **보안 계층** : JWT 유효성 검사, OAuth2지원, API 키, DDoS 공격 제어- 이 모든 긴능을 한 곳에서 제공, 웹훅 폭주 공격으로부터 벡엔드 여러번 보호
3. **페이로드 정리** : 수신되는 웹훅이나 타사 API 는 복잡한 형태를 띄는 경우가 많은데 타사의 API 가 자사의 서비스에 도달하기 전에 페이로드를 깔끔하고 스키마를 준수하는 JSON 형식으로 사전 처리하는 작업 수행
4. **모니터링 및 버전 관리** : 어떤 엔트포인트에서 노이즈 발생하는지, 어떤 버전이 최신 버전이 아닌지 정확히 파악 가능

## 로드 밸런서 : 트래픽 교통경찰

- 사용자가 누구인지 관심 없고 어떤 도로도 정체되지 않도록 관리

### 특징

1. 부하 분산 : 라운드로빈, 최소 연결 방식 또는 사용자 지정 로직을 통해 모든 서버가 안정적으로 작동. 트래픽이 급증하는 경우에도 특정 인스턴스 문제가 발생하지 않음
2. 상태 정검 : 충돌 발생 가능하지만 LB 가 이를 감지하고 자동으로 경로를 재설정
3. 자동 스케일링 통합 : 클라우드 네이티브 환경에서 자동 스케일러와 긴밀하게 연동. 별도의 조치 없이도 확장성 확보할 수 있음

## 사용 경우!

### API 게이트웨이 사용해야 하는 경우

- 외부 API 또는 웹훅을 노출
- 보안, 버전 관리 또는 변환 로직을 서비스에서 분리하고 싶은 경우
- API 사용량에 대한 속도 제한, 페이로드 검증 또는 분석 기능 필요
- **정밀한 트래픽 제어**(쓰로틀링&쿼터): 세밀한 Rate Limiting 필요할 때
- **복잡한 인증/인가**: Cognito, IAM 은 물론 커스텀 인증 로직을 붙혀 요청이 백엔드 서버 닿기 전에 JWT 토큰 유효성을 엣지단에서 검사할 수 있음
- **서버리스 아키텍처** : 백엔드가 AWS Lambda 함수들로 쪼개져있으면 HTTP 엔드포인트로 노출시켜주는 API Gateway는 선택아닌 필수
- **데이터 변환** : 클라이언트의 데이터 형태를 백엔드가 원하는 포맷으로 조작해 넘겨줄 수 있음

### 로드 벨런서를 사용해야 하는 경우

- **초고속/대용량 트래픽** : 수평 확장이 가능한 서비스를 보유한 경우, ALB 는 APIGW 보다 지연시간이 짧고 비용이 압도적 저렴
- 시스템 가동시간 자동 복구, 지연시간 기반 라우팅에 관심이 있는 경우
- 자동 확장 및 복원력 있는 아키텍처를 최대한 활용하고 싶은 경우
- **내부 통신** : VPC 내부에서 EKS 파드들이나 EC2 간 통신할 때
- **레거시 웹 서버** : 순수한 HTML/CSS/JS 서빙하거나 무거운 파일 업로드/다운로드 필요한 환경에 적합

### 둘 다 사용하는 경우

- 대부분은 제어 및 보안을 위해 API 게이트웨이를 앞에 두고 성능 및 안정성을 위해 로드밸런서를 뒤에 배치, 이는 과도한 설계가 아니며 베스트 프랙티스

### 트래픽 플로우 정리

1. 사용자가 [hyendai.com/api/users](http://hyendai.com/api/users) 요청
2. CloudFront가 /api/* 규칙 확인해 API Gateway(오리진)로 토스
3. API Gateway 가 권한 검사를 마친 후, VPC Link라는 프라이빗 터널을 타고 VPC 내부로 진입
4. 내부 ALB가 트래픽을 받아 EKS 내부 파드들로 라우팅

## 개발 팁

### 게이트웨이 해킹:

- 경로기반 라우팅을 활용해 마이크로서비스 업데이트 간소화
    - AWS API  Gateway 또는 Traefik 과 같은 API 게이트웨이를 사용해 /v1은 기존 경로 /v2는 새서비스로 라우팅하면 다운타임 없이 간편하게 배포 및 롤백 가능
- 불안정한 공급업체를 위해 서킷브레이커 설정
    - 신뢰할 수 없는 제 3자로부터 오는 웹훅은 시스템에 심각한 장애를 발생시킬 수 있음
    - Apigee 장애 규칙이나 Envoy 회로 차단기 같은 도구를 사용하면 오류 발생 엔드포인트 요청을 일시 중지하여 연쇄 장애 차단할 수 있음
- 마이크로서비스 효율적으로 유지하려면 게이트웨이에서 인증 처리 담당
    - OAuth2 모듈 사용하는 NGINX 등을 사용해 토큰 유효성 검사를 처리하면 서비스는 비즈니스 로직에만 집중할 수 있음
